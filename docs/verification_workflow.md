# Verification Workflow

Use this workflow to prove that the PySide6 conversion matches the original workbook.

## 1. Use The Same Inputs

Start with the original `MONOFUNI.xls` and the PySide6 app loaded from the same project folder.

Run the app from the repository root:

```bash
.venv/bin/python -m winsif_mon.main
```

The first parity target is the input model, not the full solver:

- `General Plant Data` must match workbook sheet `F01`.
- `Terrain Profile` must match workbook sheet `F02`.
- `Support Geometry` must match workbook sheet `F03` / VBA class `Linea`.
- `Span Geometry` must match workbook sheet `F04` / VBA class `Linea1`.
- `Custom Load Cases` must match workbook sheet `F06`.
- `Friction And Drawing Tools` must match workbook sheet `F07`.

Change one small value at a time in both systems. Good first values are `working_speed`, `rated_tightener_tension`, one terrain elevation, and one support coordinate.

## 2. Compare Outputs

In Excel, run the matching calculation button from `F05`. Then compare these workbook sheets with the PySide6 result pages:

- `Line Results` against `F08`
- `Verification Plots` against `F09`
- `Maximum And Minimum Results` against `F10`
- `Power Summary` against `F11`
- `General Report` against `F12`
- `Laying Tension` against `F20`

At the moment, these PySide6 pages load workbook defaults and preserve the visible table structure. They are not yet generated by a fully ported Python version of `Modulo1.esegui_verifica`, so final numerical solver parity is still pending.

## 3. Golden-File Test Method

For reliable conversion, export each Excel output sheet after a known calculation to CSV files:

```text
verification/excel/F08.csv
verification/excel/F09.csv
verification/excel/F10.csv
verification/excel/F11.csv
verification/excel/F12.csv
verification/excel/F20.csv
```

After the Python solver is ported, the app should export its generated results to:

```text
verification/pyside/F08.csv
verification/pyside/F09.csv
verification/pyside/F10.csv
verification/pyside/F11.csv
verification/pyside/F12.csv
verification/pyside/F20.csv
```

The acceptance rule should be exact text match for labels and units, and numerical tolerance for calculated values. A practical engineering tolerance is `1e-6` for raw values unless the Excel sheet rounds the displayed number.

## 4. What Still Blocks Full Parity

The workbook data is readable. The missing work is porting the calculation core from `VBA_Export_20260720_120646/Modules/Modulo1.bas`, especially:

- `assegna_generali`
- `leggi_terreno`
- `leggi_linea`
- `esegui_verifica`
- `analisi`
- `stampa_maxmin`
- `stampa_potenze`
- `rel_gen`

The external tools `MF_CREA_WINDXF.EXE` and `view_line.exe` are not present, so DXF generation and the old external viewer cannot be verified exactly from this workspace.
